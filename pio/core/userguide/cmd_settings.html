SettingsÂ¶

check_platformio_intervalÂ¶

Default

7

Values

Days (Number)

An interval for checking for the new version of PIO Core.

check_prune_system_thresholdÂ¶

Default

1024

Values

Megabytes (Number)

Check for pruning unnecessary data threshold (megabytes). You can list unnecessary PIO system data using pio system prune --dry-run command. See pio system prune for the other options.

Do disable checking for unnecessary data, please set threshold to 0 using pio settings set command.

enable_cacheÂ¶

Default

Yes

Values

Yes/No

Enable caching for Package Manager and for API requests.

force_verboseÂ¶

Default

No

Values

Yes/No

Force verbose output when processing environments. This setting overrides

  • pio run --verbose

  • pio ci --verbose

  • pio test --verbose

projects_dirÂ¶

Default

~/Documents/PIO/Projects

Values

Path to folder

Default location for PIO projects (PIO Home).

enable_proxy_strict_sslÂ¶

Default

Yes

Values

Yes/No

Verify the proxy server certificate against the list of supplied CAs. See also PIO Core Proxy Configuration.

enable_telemetryÂ¶

Default

Yes

Values

Yes/No

Share minimal diagnostics and usage information to help us make PIO better.

The source code of telemetry service is open source. You can make sure that we DO NOT SHARE PRIVATE information or source code of your project. All information shares ANONYMOUSLY.

Which data do we collect and why?

  • A version of Python Interpreter. PIO Core (CLI) is written in Python language, including development Platforms. We need to know which Python version produces such type of exceptions (see below), which is more popular, which version we should drop and focus on a new one

  • PIO Core (CLI) errors/exceptions. We report automatically fatal exceptions raised by PIO Core source code but NOT by your project

  • The name of the used platform, board, framework. We collect this type of information to have a clear picture which software products are the most widely used by our Community and for the which we should provide frequent updates and add new features ( for example, âatmelavrâ, âarduinoâ, âunoâ, etc.)

  • The name of CLI command. It helps us to improve our CLI. For example, ârunâ, âlib listâ)

  • The name of Cloud & Desktop IDEs. This is very important information for us. We create native extensions based on the popularity of IDEs (for example, PIO IDE for VSCode)

Thanks a lot that you keep this setting enabled!
