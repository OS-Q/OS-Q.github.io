InstallationÂ¶

Note

Please note that you do not need to install PIO Core (CLI) separately if you are going to use PIO IDE for VSCode. PIO Core (CLI) is built into PIO IDE and you will be able to use it within PIO IDE Terminal.

If you are going to use Git for installing upstream development platforms, cloning external projects, installing library dependencies from a repository, please ensure that git --version command works from a system terminal. Otherwise, please install a Git client.

  1. Download and install official Microsoft Visual Studio Code. PIO IDE is built on top of it

  2. Open VSCode Package Manager

  3. Search for the official platformio ide extension

  4. Install PIO IDE.

Setting Up the ProjectÂ¶

  1. Click on âPIO Homeâ button on the bottom PIO Toolbar

  1. Click on âNew Projectâ, select a board and create new PIO Project

The code below works only in pair with Arduino-based boards. Please follow to PIO Project Examples repository for other pre-configured projects.

/**
 * Blink
 *
 * Turns on an LED on for one second,
 * then off for one second, repeatedly.
 */
#include "Arduino.h"

// Set LED_BUILTIN if it is not defined by Arduino framework
// #define LED_BUILTIN 13

void setup()
{
  // initialize LED digital pin as an output.
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop()
{
  // turn the LED on (HIGH is the voltage level)
  digitalWrite(LED_BUILTIN, HIGH);

  // wait for a second
  delay(1000);

  // turn the LED off by making the voltage LOW
  digitalWrite(LED_BUILTIN, LOW);

   // wait for a second
  delay(1000);
}

Task RunnerÂ¶

PIO IDE provides built-in tasks through the menu Terminal > Run Task... (Build, Upload, Clean, Monitor, etc) and custom tasks per âplatformio.iniâ (Project Configuration File) environment ([env:***]). The default behavior is to use Terminal Panels for presentation, one panel dedicated to each unique task.

The PIO IDE provides its own Problems Matcher named $platformio. You can use it later if you decide to change base task settings.

You can override existing tasks with your own presentation options. For example, letâs configure PIO Task Runner to use a NEW Terminal panel for each âBuildâ command:

  1. The menu item Terminal > Run Task... opens up a list of VSCode tasks for PIO. In the line PIO: Build, press the gear icon on the far right side of the list. This creates or opens the file .vscode/tasks.json with some template code.

  2. Replace the template in tasks.json with this code

{
    "version": "2.0.0",
    "tasks": [
        {
            "type": "PIO",
            "task": "Monitor",
            "problemMatcher": [
                "$platformio"
            ],
            "presentation": {
                "panel": "new"
            }
        }
    ]
}

See more options in the official VSCode documentation.

Serial Port MonitorÂ¶

You can customize Serial Port Monitor using Monitor options in âplatformio.iniâ (Project Configuration File):

  • monitor_filters

  • monitor_rts

Example:

framework = arduino
board = esp32dev

; Custom Serial Monitor port

; Custom Serial Monitor speed (baud rate)

DebuggingÂ¶

Debugging in VSCode works in combination with Debugging. You should have PIO Account to work with it.

VSCode has a separate activity view named âDebugâ (accessed by the bug icon on the left toolbar). Debugging extends it with more advanced debugging instruments and features:

  • Local, Global, and Static Variable Explorer

  • Conditional Breakpoints

  • Expressions and Watchpoints

  • Generic Registers

  • Peripheral Registers

  • Memory Viewer

  • Disassembly

  • Multi-thread support

  • A hot restart of an active debugging session.

There are two pre-configured debugging configurations:

PIO Debug

Default configuration. PIO runs the Pre-Debug task and builds the project using Debug Configuration. Also, it checks for project changes.

PIO Debug (skip Pre-Debug)

PIO skips the Pre-Debug stage and DOES NOT build or check the project for changes. If you do changes in project source files, they will not be reflected in debug sessions until you switch back to the âPIO Debugâ configuration or manually run the âPre-Debugâ task.

This configuration is very useful for quick debug session. It is super fast by skipping several checks, letting you control project changes manually.

Note

Please note that Debugging will use the first declared build environment in âplatformio.iniâ (Project Configuration File) if the default_envs option is not specified.

Variable FormatÂ¶

Currently, VSCode does not provide an UI or API to change the variable format. See the related VSCode Issue #28025.

A temporary solution is to set the default numerical base in which the debugger displays numeric output in the Debug Console. (The Debug Console is visible during active debugging sessions). For example, to show variables in hexadecimal format, copy the code below and paste it into âDebug Consoleâ:

set output-radix 16

Possible values, listed in decimal base, are: 8, 10, 16.

WatchpointsÂ¶

Please read GDB: Setting Watchpoints first.

Currently, VSCode does not provide an API to change the value format of watchpoints. You can manually cast watchpoint expressions to display the value as specific pointer types:

  • $pc, default decimal integer format

  • *0x10012000, an address, default decimal integer format

  • (void*)$pc, $pc register, hexadecimal format

  • *(void**)0x10012000, an address, hexadecimal format

SettingsÂ¶

How to configure VSCode settings?

platformio-ide.activateOnlyOnPlatformIOProjectÂ¶

If true, activate the platformio ide extension only when a PIO-based project (that has a âplatformio.iniâ (Project Configuration File)) is open in the workspace. The default value is false.

platformio-ide.activateProjectOnTextEditorChangeÂ¶

Automatically activate project depending on an active opened text editor. The default value is false.

platformio-ide.autoOpenPlatformIOIniFileÂ¶

Automatically open the âplatformio.iniâ (Project Configuration File) file from a project when no other editors are opened. The default value is true.

platformio-ide.autoCloseSerialMonitorÂ¶

If true, automatically close pio device monitor before uploading/testing. The default value is true.

platformio-ide.autoRebuildAutocompleteIndexÂ¶

If true, automatically rebuild the C/C++ Project Index when âplatformio.iniâ (Project Configuration File) is changed or when new libraries are installed. The default value is true.

platformio-ide.buildTaskÂ¶

The build task (label) that is launched by the âBuildâ button in the PIO Toolbar and Key Bindings. The default is PIO: Build.

You can create custom Custom Tasks and assign one of them to platformio-ide.buildTask.

platformio-ide.autoPreloadEnvTasksÂ¶

Automatically preload ALL project environment tasks. The default value is false.

platformio-ide.customPATHÂ¶

Custom PATH for the platformio command. Paste here the result of echo $PATH (Unix) / echo %PATH% (Windows) command by typing into your system terminal if you prefer to use a custom version of PIO Core (CLI). The default value is null, meaning PIO looks for the platformio command in the system path.

platformio-ide.disableToolbarÂ¶

Disable the PIO toolbar. The default value is false.

platformio-ide.forceUploadAndMonitorÂ¶

If true, the Upload (platformio-ide.upload) command is changed to use the âUpload and Monitorâ task. The default value is false.

platformio-ide.reopenSerialMonitorDelayÂ¶

Configure the time in milliseconds before reopening the Serial Port Monitor. The default value is 0, which means to reopen instantly.

platformio-ide.useBuiltinPythonÂ¶

Use a portable Python 3 Interpreter if available. The default value is true.

platformio-ide.useBuiltinPIOCoreÂ¶

If true, use the built-in PIO Core (CLI). The default value is true.

platformio-ide.useDevelopmentPIOCoreÂ¶

If true, use the development version of PIO Core (CLI). The default value is false.

platformio-ide.disablePIOHomeStartupÂ¶

Disable showing PIO Home at startup. The default value is false.

platformio-ide.pioHomeServerHttpHostÂ¶

PIO Home server HTTP host.The default is 127.0.0.1, but in case of dockerized environments 0.0.0.0.

platformio-ide.pioHomeServerHttpPortÂ¶

PIO Home server HTTP port. The default value 0 automatically assigns a free port in the range [8010..8100]).

platformio-ide.customPyPiIndexUrlÂ¶

Custom base URL of the Python Package Index (default https://pypi.org/simple).
